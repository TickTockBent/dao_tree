# Design principles

Wes's standing principles, banked across the 2026-07-02 review passes. These
are the project's design constitution: when a proposal touches one's
territory, check it against the principle before recommending. Organized by
theme (the pass-by-pass history is in git).

## Narrative and information

1. **Chronicle + knowledge veil are duals, designed together.** Past legible
   as narrative, future illegible as mystery — one posture: "memory accretes,
   foresight is earned."
2. **Veil the ahead, never the now.** The current loop's optimization
   language (grades, rates) is never hidden — incremental players are
   optimizers first and read it as hiding the ball. Mystify only
   flavor-critical, optimization-optional horizon content.
3. **Watch scope anchoring.** Don't price a feature at its floor ("data table
   + formatter") when its ceiling is a pillar. Decide flavor-line vs system
   consciously; the cheap version forecloses the tall one.

## Build diversity and balance

4. **Counter-monopoly test.** A system lock is fine only if each alternative
   owns something equally identity-defining. Final triad: lattice owns WHO,
   pill owns throughput, sect owns THE WORLD (between lives).
5. **The sim measures time; some builds own non-temporal axes.** A trailing
   build isn't under-tuned if it owns an axis the sim can't price. Flip
   condition: trailing AND owning nothing distinct = under-tuned.
6. **Attribution before action.** Counterfactual-probe the suspected cause
   before changing anything — the method that exonerated the aspect lock and
   found the real trap.
7. **Findings are relational.** Run variants side-by-side through both
   actors; a single variant tested alone reads "faster, good." The
   multi-variant grid is what reveals inversions.

## Grind, ritual, and pacing

8. **Ritual vs tax.** When removing emergent grind, the alternative to
   deletion is making it FELT. Same code touch, opposite philosophies —
   choose deliberately; never sleepwalk into the delete.
9. **Load-bearing-structure test.** Nothing that is *only* a tax accidentally
   becomes ~¾ of optimal play. Magnitude that large is structure — deleting
   it kicks out a wall and relocates the rebuild somewhere more expensive.
10. **No static beat survives dominant repetition.** Ritualizing a large
    time-share requires a CHANGING payoff — a curve the player feels bend.
11. **Probe curves, not totals, when the mechanic is the spine.** A uniform
    discount and an acceleration can land the same hours and are opposite
    designs — one is a smaller tax, the other is felt mastery.
12. **Brief ritual, never vanished.** Mastery curves asymptote to a felt
    minimum ("a breath"), not zero — a vanished ritual is full-keep with
    extra steps. Judge tails by single-breath counts.
13. **Match curve complexity to sample count.** Twelve re-climbs support
    "noticeably faster each time," not "fast, plateau, fast again."
14. **Thin-curve fallback = extend across acts**, never more churn.

## Calibration and measurement

15. **Decompose before pinning.** Never pin a band containing a fixable
    inefficiency; bands wait on the decisions that move them.
16. **Jitter-test calibration constants.** Narrow swing → pin a point; wide
    swing → pin a range ("53.25 is a number vs a mood").
17. **Pin the assumption, not just the number.** A cadence-shaped band is
    pinned WITH its cadence label; record the revival condition for dead
    knobs.
18. **Band width is information.** A wide experience band means the game
    accommodates multiple session cadences — report as a design fact, never
    as tolerated imprecision.
19. **The optimizer floor is an instrument, not a claim.** Once an
    experience-target actor exists, defend the Realistic band at planning
    time; pin the optimizer for regression detection.

## Economy and meta-structure

20. **Unbounded meta-currencies are explicit decisions.** Eternal scope +
    farmable source = unbounded across lives; the economy answers
    bounded-vs-unbounded FIRST, before numbers.
21. **Wider, not taller.** World-state accumulation: diminishing marginal
    returns per AXIS, unbounded BREADTH (always new categories; local →
    regional → continental frontier). Flat caps kill the fantasy; uncapped
    totals recreate the Diligent problem at world scale.
22. **A prestige loop converts FOMO into loot.** Rarity that creates
    permanent inequality in a single-run frame becomes "one more run" in a
    loop frame — Samsara licenses dramatic rare events.
23. **Loot, never gate** (sim-enforced). Base completion never requires a
    rare event; rare events change a run's CHARACTER, not whether it
    finishes.
24. **Aggregate-certain, specific-rare.** Every life near-certainly
    encounters SOMETHING unusual; no life encounters everything. A life
    where nothing unusual happened is a system failure.
25. **Two tiers, two amplitudes, two registers.** Predictable cycle moderate
    (1.3–2×, rewards planning); rare spawns dramatic (3–10×, rewards
    presence, creates stories). Don't mix the registers.
26. **Endogenous uncertainty beats hidden RNG.** The best bets show
    everything and make the player's own future pace the unknown (the
    ginseng principle). Prefer decision-creating rare events over
    window-shaped ones; lean: loot rolls on engagement, not wall-clock.
27. **Acceleration constants are genre decisions.** An r-like decay factor
    encodes fantasy posture (0.85 = the game respects the obstacle; 0.5 = it
    celebrates distance traveled). Suspect any fixed constant that fails at
    either end — it may need to be a function.

## Architecture and process

28. **Accumulators are typed, not bespoke.** One engine concept, scope enum
    (soul | world | dao-reserved); instances are data entries. Define the
    type at the first instance, generalize at the second scope, validate at
    the third — "leave room" is an enum value and a paragraph, never
    speculative code.
29. **Convergent design beats additive design.** When five planned features
    turn out to be five expressions of one system, build the system.
30. **Build sequencing is the dynasty metagame.** With soul- and
    world-scoped accumulators, lives are MOVES in a save-file-scale
    strategy; the chronicle records the strategy, not the stats.
31. **Multi-life harness before world-rate tuning.** Single-life sims are
    structurally blind to cross-life effects.
32. **Invert survival lists when they grow.** Define what the reset RESETS
    and let persistence be the default, once eternal-scoped state stops
    being exceptional.
33. **Ambient before interactive** (rivals and beyond): ship the
    world-feels-inhabited layer at fraction cost; architect hooks for
    interaction later.
34. **Steam soft-yes discipline**: save-migration and achievement
    scaffolding are cheap to build in and expensive to retrofit — option
    value decays fastest of all parked decisions.
